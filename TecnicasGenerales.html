Tecnicas generales

G217 Permitir al usuario modificar o desactivar los atajos

Algunos autores crean atajos en sus aplicaciones para permitir a los usuarios interacciones más rápidas. Pero cuando los atajos solamente están formados por un carácter como clave, como pueden ser letras, números, o símbolos, sin modificadores como por ejemplo la tecla control, crean unas dificultades para los usuarios que funcionan con el uso por voz, además de usuarios con dificultades para escribir de forma precisa.

Para evitar activaciones involuntarias, los creadores deben permitir a los usuarios poder desactivar los atajos o reconfigurar las teclas para dichos atajos. Configurar el atajo debe permitir no solo modificar la tecla, sino el uso de teclas modificadoras.

G216 Proporcionar activación de un solo punto para un selector deslizante

El objetivo de esta técnica es asegurar que los usuarios que tienen dificultades para realizar movimientos rectos y precisos puedan controlar con facilidad y sin problemas las barras control, como por ejemplo una barra para controlar el volumen.

Una forma de ayudar a los usuarios de forma simple, es dar la posibilidad de hacer click en la posición a la que quieres que se desplace. Así se puede asignar el valor con un solo click.

Dar controles alternativos como por ejemplo moverlo con las flechas del teclado.


3-G215 Proporcionar controles para lograr el mismo resultado que los gestos multipunto o basados ​​en ruta

El objetivo es asegurar que los usuarios que tienen dificultades a la hora de utilizar desplegables puedan operar con un solo click. Un desplegable de contenido contiene bloques de contenido dentro de una fila. En ocasiones varios de estos bloques de contenido se encuentran ocultos y solo uno de ellos es visible a cada momento. Deslizar por encima del bloque visible hace que los bloques contenidos ocultas se vuelvan visibles. Es importante dar al usuario herramientas alternativas como por ejemplo las flechas para poder desplazarse por estos menús.

En dispositivos de pantallas táctiles, estas funciones no suelen funcionar cuando a nivel de sistema operativo hay tecnología de asistencia como por ejemplo un lector de pantalla activado.

G214 Usar un control para permitir el acceso al contenido en diferentes orientaciones que de otra manera estaría restringido

El objetivo de esta técnica se trata de permitir a los usuarios acceder al contenido de la manera que el usuario prefiera. Desde el punto de vista de quien crea el contenido, es normal esperar que el usuario vaya a verlo utilizando una orientación específica o que va a querer mantener la visión original en el dispositivo. Como resultado se evita que el contenido pueda rotar, pero proveyendo al usuario con una herramienta que permita al usuario rotar el contenido, si alguien que tiene una necesidad de utilizar una orientación específica podrá ver el contenido de forma cómoda. Por ejemplo alguien que no puede sostener el dispositivo y lo tiene unido a una silla de ruedas o la coma.

Un ejemplo de codigo que te permite girar la orientación de la página seria el siguiente en CSS.

#container { display:block; }
            @media only screen and (orientation:portrait){
            #container { 
                    -webkit-transform: rotate(90deg);
                    -moz-transform: rotate(90deg);
                    -o-transform: rotate(90deg);
                    -ms-transform: rotate(90deg);
                    transform: rotate(90deg);
                    }
    
            }
            @media only screen and (orientation:landscape){
            #container {  
                    -webkit-transform: rotate(0deg);
                    -moz-transform: rotate(0deg);
                    -o-transform: rotate(0deg);
                    -ms-transform: rotate(0deg);
                    transform: rotate(0deg);
                    }
            }

G213 Proporciona controles convencionales y una configuración de aplicación para la entrada activada por movimiento

El objetivo de esta técnica es la de asegurar:

Personas que utilizan dispositivos acoplados o que no pueden utilizar de forma precisa los dispositivos, sean capaces de inputs de forma más fácil a través de componentes de interfaz.
Personas que podrían activar de forma accidental los sensores debido al temblor o a otros impedimentos debido al motor puedan desactivar las acciones de movimiento.

Cuando un sensor como por ejemplo un giroscopio es utilizado para entrada:

Dar otras opciones que no dependan del sensor, tales como botones o enlaces por ejemplo.
Dar la posibilidad al usuario para desactivar el sensor.

G212 Usar controles nativos para garantizar que la funcionalidad se active en el evento up.

El objetivo de esta técnica es asegurar que los usuarios que intenten interactuar con un control, no activen el evento de forma accidental. Esto puede conseguirse con el uso de activadores del evento.

El método más sencillo para conseguir este criterio es utilizar los comportamientos por defecto de los controladores, no sobreescribiendo su comportamiento.

En lenguajes nativos donde el control es activado en un evento inactivo, es normalmente por un buen motivo y fácilmente reversible.

G211 Hacer coincidir el nombre accesible con la etiqueta visible

Cuando usuarios que utilizan una entrada por voz para interactuar con la página web, normalmente dicen un comando seguido por la referencia de alguna etiqueta visible. Por ejemplo, ellos podrían decir, “click en buscar” para acceder al apartado de búsqueda. Cuando el programa de reconocimiento de voz recibe la entrada, busca por ocurrencias en los nombres de los controles. Donde haya una diferencia entre el texto en la etiqueta y el texto en el nombre puede causar problemas para el usuario. La forma más simple de activar la entrada por voz es hacer que el nombre de accesibilidad sea el mismo que el del texto visible.

Algunas veces más de una cadena de texto se encuentra alrededor de un control y puede ser candidata para la etiqueta. Por ejemplo una serie de entradas con su propia etiqueta debe ir precedida por una cabecera, una instrucción o un grupo de etiquetas (Como por ejemplo HTML legend /fieldset). Es recomendable que solo se utilicen etiquetas cercanas para el propósito de ser etiquetas de control.

G210 Garantizar que las acciones de arrastrar y soltar se puedan cancelar

El objetivo de esta técnica trata de que el usuario pueda cancelar la acción de arrastrar un objeto y soltarlo en otro punto. Esto puede ser hecho soltando el objeto fuera de la zona donde está preparada para que sea soltada o moviendo el objeto a la posición en la que se encontraba. Otra opción posible es la de una vez arrastrado el objeto pedir una confirmación de si se quiere realizar la acción.

En pantallas táctiles cuando están activados programas para la lectura de pantalla, puede ocurrir que al arrastrar lo que ocurra será que se desplace únicamente el foco del programa de lectura.

G209 Proporciona suficiente contraste en los límites entre colores contiguos

El objetivo de esta tecnica se trata de que personas con problemas de vista moderados pueden distinguir los bordes entre distintos segmentos de color.

El criterio de éxito para el contraste sin texto utiliza el término "objeto gráfico" para cubrir pequeños gráficos simples y partes de gráficos complejos más grandes. Esta técnica se enfoca en segmentos de colores sólidos donde los límites entre colores transmiten significado.

Cuando elegimos un color para gráficos con múltiples colores, al añadir colores adyacentes comprueba que al menos el contraste es de 3:1. En caso de que no se cumpla añade un borde que contraste al menos en 3:1 entre ellos.

G204 No interferir con reposicionamiento del texto cuando la ventana de visualización se reduce

Esta técnica ayuda a evitar situaciones donde el scroll horizontal pudiera ocurrir. Muchas personas con problemas cognitivos y baja visibilidad que no utilizan tecnología de asistencia tienen grandes problemas a la hora de lidiar con bloques de texto en los que debes utilizar el scroll horizontal. Una de las mejores formas de hacer esto es definir un ancho para el contenedor del bloque de texto con porcentajes.

HTML y XHTML utiliza agentes que automáticamente recoloca el texto en cuando la ventana del navegador es achicada siempre y cuando el autor no especifique ancho con medidas absolutas como pixeles.
